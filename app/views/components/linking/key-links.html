Key links

The key links list is a navigation component that is used on a sub-topic hub to display all the related content pages within that sub-topic.

image

code example

When to use the key links

Key links should be used on sub-topic hub pages, but should not be used on the main topic hub pages.

Key links should be used when there is more than one link within the sub-topic. If only one link exists, then the main topic hub should link directly to that content page, rather than adding an additional step.

How to use the key links

The links should sit under the H1 and any lede text.

There should be a link to every content child page within the sub-topic hub.

Links should be styled consistently with other hyperlinks on the site - they should be blue and underlined.

Once a user has viewed a page using the key links, the link should be styled as a visited link.

Benefits of the key links

Research

During testing, NHS.UK's Nav Team team found that the key links component gave users an overview of all the content they would be able to find within each sub-topic, helping them to evaluate where they were in the site, and if it was the right place to meet their needs.
