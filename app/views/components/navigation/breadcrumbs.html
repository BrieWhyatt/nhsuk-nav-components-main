Breadcrumbs

Breadcrumbs show users where they are in a website's structure and how to get all the way back to the homepage.

image

code example

When to use breadcrumbs

Breadcrumbs should be used on every content page, such as a health information page.

This helps users understand where they are in the website's hierarchical structure and how to back through related content.

How to use breadcrumbs

Breadcrumbs should be placed at the top of a content page, below the header navigation and above the H1 of the page.

They should show the hierarchy of content pages starting at the homepage and ending at the parent of the page the user is on. For example, on the Chickenpox page, the breadcrumb will read:

Home > Health A to Z

and then the H1 of the page would read: Chickenpox

Breadcrumbs should not include a link to the page a user is currently on, the H1 of the page is there to let users know they're in the right place.

Breadcrumbs on mobile screens

On a mobile screen size, the breadcrumb will just show the parent page, with a back arrow, like this:

< Health A to Z

The breadcrumbs should be styled consistently with the rest of the hyperlinks on site, so for NHS.UK we use blue link text, with a blue underline.

Accessibility

Breadcrumbs should not include a link to the page a user is currently on. In research, users with screen readers have found long breadcrumb tails and long H1s problematic when read out together.

Benefits of breadcrumbs

Research

During testing, NHS.UK's Information Architecture team found users were confident using breadcrumbs, and often used them to go up a level and explore what other content was available.
